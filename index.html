SuperResolution을 이용한 User classification

팀 프로젝트(학과) / 4개월 / github


    문제점
  • 익명성이 높은 다크넷 마켓의 다중계정 문제로 인한 유저 파악의 어려움
  • Photograph style analysis를 활용한 유저 분류를 사용할 때 다크넷 마켓에 업로드된 저화질 이미지로 인하여 기존 분류 모델의 성능이 떨어질 것으로 예측

    해결책
  • Pre-trained DNN 모델을 이용하여 다크넷 이미지로 Fine-tuning을 통해 User classification
  • Super-Resolution 모델을 사용하여 다크넷 이미지의 화질 향상 후 DNN 모델을 이용하여 학습

    수행한 것
  • 다크넷의 크롤링 데이터에서 이미지를 추출하여 유저별로 분류하는 Pre-processing 과정을 통해 Input dataset 생성
  • ResNet18 모델에 다크넷 이미지 dataset을 넣고 학습시킨 You are your photographs 논문 구현
  • SRCNN(Image Super-Resolution Using Deep Convolutional Networks) 논문을 구현하여 Super-Resolution을 통한 이미지 화질 향상
  • Super-Resolution을 적용한 다크넷 이미지를 ResNet18의 Fine-tuning dataset으로 사용 후 학습
  • 기존 논문과 Super-Resolution을 적용한 DNN 모델 간의 User classification 정확도 비교

    배운 것
  • Python의 입출력, 정규표현식 모듈을 활용한 데이터 전처리 및 분석
  • Python의 딥러닝 모듈 Pytorch를 활용한 SRCNN, RESNET모델 구현, 실험

RF를 통한 경기순환시계 예측, 투자 모델 제안

팀 프로젝트(공모전) / 6개월 / github


    문제점
  • 주식, 채권, 원자재, 금으로 구성된 All Weather Portfolid(AWP)는 경기 국면에 무관하게 수익을 낼 수 있는 자산을 보유한다는 장점이 있지만 고정된 비중으로 투자하기 때문에 수동적인 포트폴리오라는 단점이 존재
  • AWF는 미국에서 시작된 포트폴리오이기 때문에 한국의 상품을 기준으로 고객에 따라 커스터마이징된 포트폴리오는 없음

    해결책
  • 경기순환시계 지표를 통해 4가지 국면(상승, 하강, 둔화, 회복)을 분류하고 다음 달의 국면을 예측하는 모델을 설계하여 이를 기반으로 포트폴리오의 국면별 최적 자산 비중 산출
  • 고객데이터를 기반으로 타겟 고객을 선정하고 고객별 위험 선호도, 자산 선호도, 시가총액 선호도를 기반으로 ETF로 구성된 AWF를 제안

    수행한 것
  • 통계청의 경기순환시계의 경제지표 좌표를 크롤링
  • RandomForest Linear Regression model을 통해 과거 12개월 경기순환시계 좌표 데이터로부터 1달 후의 데이터를 예측
  • K-means clustering을 통해 해당 월의 경기 국면을 분류
  • 클러스터링 된 경기 국면을 토대로 기간을 4개로 나눈 후 Search algorithm을 통해 국면별 포트폴리오 최적 자산 비중을 계산
  • 고객데이터를 통해 AWF 포트폴리오 타겟 고객을 클러스터링하고 타겟 고객의 선호도를 분석하여 포트폴리오 구성 ETF와 자산 비중 제안

    배운 것
  • 크롬 개발자도구와 JavaScript를 활용한 웹 크롤링
  • Python 입출력, Pandas 모듈을 활용한 데이터 전처리 및 분석
  • Python 머신러닝 모듈 Sklearn을 활용한 RandomForest, K-means clustering 활용

Unity를 사용한 Shooting Game 만들기

개인 프로젝트 / 6개월 / github


스테이지와 Player, Enemy, Boss가 등장하는 일반적인 슈팅 게임을 모작하였습니다. 게임을 기획하고 게임 로직을 Unity와 C#을 통하여 구현하였으며, 게임 내 UI도 구현하였습니다.

    수행한 것
  • Player 구현 (Move, Fire, Boom, 다른 객체와의 충돌 등)
  • Enemy 구현 (Move, Fire, 다른 객체와의 충돌 등)
  • Boss 구현 (랜덤을 활용한 4가지 패턴)
  • GameManager 구현 (전반적인 게임 관리, UI 관리 등)
  • ObjectManager 구현 (ObjectPooling을 통한 관리)

    배운 것
  • C#의 기본적인 문법과 객체지향프로그래밍의 개념
  • Unity를 활용한 객체 간 참조, 컴포넌트 참조 등의 Unity 플랫폼 사용법
  • 디자인패턴(ObjectPool, Singleton)
